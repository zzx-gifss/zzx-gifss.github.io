Ajax 不需要刷新页面就能发送和获取信息，能使网页实现异步更新。

发送请求的五个步骤

  1. 创建异步对象

  2. 设置请求参数 包括请求的方法 URL

  3. 发送请求

  4. 注册事件 onreadystatechange事件

  5. 获取返回的数据

正常的href请求和ajax请求的区别

Href只是一个链接，不需要js的参与。也不需要和原网页进行通信

Ajax是在HTML和服务器中间搭桥，由于HTML和服务器直接通信需要刷新界面，而由js和服务器的通信并由js根据接收的数据对HTML页面修改则不需要改变整个界面，所以ajax主要应用于用户和服务器的少量不需要刷新整个页面的信息交互

一般都是连接get，js可以设置并且可以加更多的header数据

Ajax请求的header里边会有xhr的标记

Xml对象：基于xml的http请求，XMLHttpRequest是一个浏览器接口，使得js可以进行http通信

AJAX: Asynchronous Javascript and xml 异步的js和xml

ajax为什么是异步的

ajax的出现就是让页面不刷新的情况下进行数据的交互，肯定是需要时间的

js的加载是从上到下的执行顺序，假如不是异步的当代码执行到ajax就会停住等待ajax的响应，而下面的js就会一直等待这个请求结束才会继续执行，网站的其他事件或者特效就会不能实现，用户体验极差。

跨域

跨域问题来源于JavaScript的同源策略，即只有 **协议+主机名+端口号 (如存在)**相同，则允许相互访问。也就是说JavaScript只能访问和操作自己域下的资源，不能访问和操作其他域下的资源。跨域问题是针对JS和ajax的，html本身没有跨域问题，比如a标签、script标签、甚至form标签（可以直接跨域发送数据并接收数据）等

加载图片css js可以无视同源策略

所有的跨域，都必须经过serve端允许和配合

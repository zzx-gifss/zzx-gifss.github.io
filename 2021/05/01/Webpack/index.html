Webpack

简介

webpack 是个静态模块打包工具。可以使用webpack将项目里所有模块利用资源依赖关系，把各模块之间关联起来。生成浏览器可以直接运行的代码。将文件打包成浏览器能看懂的文件。

通过入口文件，使用递归找到直接依赖或间接依赖的所有模块，并在构建出所有模块的依赖图，使用webpack打包生成一个或多个 bundle 文件。

  1. 可以将零散的javascript代码打包到一个js文件中
  2. 对于有环境兼容的代码，webpack可以通过loader将其实现编译转换，然后再进行打包
  3. 对于不同类型的模块，webpack支持在javascript中以模块化的方式载入任意类型的资源文件。
  4. 还可以将应用中所有的模块按需分块打包 不用担心全部代码打包到一起，导致加载慢的问题

loader和plugin

因为webpack只能解析js文件，对于非js文件，比如vue模板文件等，需要使用对应的loader将其转化为js代码。webpack还能结合对应的Plugin改变某些模板的输出结果，比如通过htmlWebpackPlugin可以对html文件的输出进行改变。

plugin的理解是插件的含义，即它是webpack打包过程中的一些插件。

使用方式

  1. 在配置文件webpack.config.js中配置
  2. 通过命令行参数配置
  3. 通过内联使用

常用loader

样式：style-loader、css-loader、less-loader、sass-loader等

文件：raw-loader、file-loader 、url-loader等

编译：babel-loader、coffee-loader 、ts-loader等

校验测试：mocha-loader、jshint-loader 、eslint-loader等

loader在module.rule中配置，作为webpack解析规则存在。Plugin在plugins中配置，每一项都是一个plugin实例，参数通过构造函数传入，下面是一个简单的使用loader和plugin的webpack文件

clean-webpack-plugin插件

在使用webpack打包时，打包后的文件名如果一样，那么每次打包后，新生成的文件就会将之前打包的文件覆盖掉，但是如果给输出的文件名设置根据内容生成的hash值后，由于每次打包后生成的文件名的hash值会不一样，这样就不会因文件名相同而覆盖原来的文件，在文件目录中出现了一些多余的文件；如果每次都手动清除这些多余文件也是很麻烦的，webpack的clean-webpack-plugin插件可以很好的帮助我们完成每次打包前的文件清除工作。

HtmlWebpackPlugin

将打包好的js文件，自动通过script标签插入到body中

常见的loader

image-loader:加载并压缩图片文件

file-loader:把文件输出到一个文件夹中，在代码中通过相对 URL 去引用输出的文件

babel-loader：把 ES6 转换成 ES5

css-loader：加载 CSS，支持模块化、压缩、文件导入等特性

style-loader：把 CSS 代码注入到 JavaScript 中，通过 DOM 操作去加载 CSS。

json-loader : 加载 JSON 文件（默认包含）

vue-loader：加载 Vue.js 单文件组件

常见的plugin

html-webpack-plugin：简化 HTML 文件创建 (依赖于 html-loader)

web-webpack-plugin：可方便地为单页应用输出 HTML，比 html-webpack-plugin 好用

clean-webpack-plugin : 删除打包文件

webpack热更新原理

当你对代码修改并保存后，将会对代码进行重新打包，并将改动的模块发送到浏览器端，浏览器用新的模块替换掉旧的模块，去实现局部更新页面而非整体刷新页面。

不使用热更新，则通过npx webpack打包

使用热更新，通过npx webpack-dev-server打包

webpack构建项目流程

初始化参数：读取配置文件，拿到参数

开始编译：根据拿到的参数，加载所有引入的插件

确定入口：找到entry中的所有文件

编译：根据loader遍历每个模块和建立每个模块之间的关系

确定出口：根据模块依赖关系，生辰不过一个包含多个模块的单独文件

输出完成：确认好输出之后，根据配置的出口生成新的文件

webpack是webpack自带的一条打包命令，是最基本的打包命令。npm run webpack，则是在package.json文件中有一个script配置项，在这个script里面自定义的一条命令，它通过npm来运行。例如：
scripts”: {
“start”: “webpack –config webpack.config.js –progress –watch”
}
当脚本名为“start”时，可以省略“run”，直接为：

npm start

你说的属于这种情况：脚本名称为“webpack”
scripts”: {
“webpack”: “webpack –config webpack.config.js –progress –watch”
}
后面可以加一些参数来方便开发，执行下面这条语句

npm run webpack
